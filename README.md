Veuillez prendre en compte les facteurs ci-dessous lorsque vous contribuez :

Style de code :
Maintenez un style de code cohérent pour favoriser la lisibilité.

Documentation :
Assurez-vous que le code soit bien documenté pour une collaboration efficace.

Tests :
Testez minutieusement vos modifications avant de soumettre une pull request.

Suivi des issues :
Consultez le système de suivi des issues pour les tâches à réaliser.

Revue de code :
Toutes les contributions sont soumises à un processus de revue de code.

Licence :
Les contributions sont soumises à une licence.
