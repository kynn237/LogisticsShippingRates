Directives de contribution

Bienvenue aux contributeurs !
Merci d'envisager de contribuer au dépôt centralisé. Ce document décrit les directives à suivre pour contribuer au développement des tarifs et calculs d'expédition.

Style de code
Veuillez suivre le style de codage et les conventions utilisés dans la base de code existante. Cela permet de maintenir la cohérence dans l'ensemble du projet.

Documentation
Assurez-vous que vos contributions soient bien documentées. Incluez des commentaires dans votre code lorsque cela est nécessaire et fournissez une description claire et concise de vos modifications dans la pull request.

Tests
Avant de soumettre une pull request, assurez-vous que vos modifications aient été testées minutieusement. Incluez des cas de test pertinents et vérifiez que les tests existants réussissent.

Suivi des issues
Consultez le système de suivi des issues pour toute tâche ou demande de fonctionnalité. Si vous travaillez sur quelque chose, veuillez commenter l'issue pour informer les autres.

Revue de code
Toutes les contributions passeront par un processus de revue de code. Soyez ouvert aux retours et prêt à faire des modifications si nécessaire. Les revues de code aident à maintenir la qualité et la cohérence du code.

Merci pour votre contribution !
